Welcome to {{ brand_name_long }}!

{{ brand_name_long }} is a collection of tools which is designed to aid the development and practical application of deep learning approaches. The mixture of available tools ranges from state-of-the-art models developed by our researchers, to openly available models and tools, which aid in the development and adaptation of our models to new problems.

Tools are designed for use individually, or as part of a guided process starting at annotation, which you can initiate with Start a tour. In the following sections, each toolset will be briefly described.

Models

The models tab contains a list of currently available models along with descriptions of what problems they were designed to address. Depending on the model, you may select it to do things such as initiate training, run inference or start a tour.

Experiments

Experiments is where your training runs will appear, and it gives you an overview of relevant training metrics, and outputs of the training process such as evaluation visualisations, raw scores or model weights.

It uses MLflow, an open-source library (and server-side application), with some adjustments to the latter for a more seamless experience in the toolbox.

Labeling

This is where you can do some labeling of your dataset if necessary, but usage will depend on your specific model or problem. For example:

  • a model like SAM3 can generate a segmentation mask based on a provided bounding box
  • models may support preannotating data
  • some models allow you to create a new labeling project with most things preconfigured
  • otherwise you might need to follow upstream documentation for additional info on configuration

It uses Label Studio, a web application for labeling with support for a wide variety of tasks, while making some adjustments for a more seamless experience in the toolbox.

Status

The status page shows the description of the current (or last) running task, along with it's log. The log may contain useful information if for example the wrong values were passed to the arguments, but for more involved errors you should contact relevant personel for assistance.
